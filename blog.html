1. What is SQL Injection?

SQL Injection is a critical vulnerability where attackers manipulate SQL queries by injecting malicious input. It can lead to unauthorized access to sensitive data.

JavaScript 

const query = "SELECT * FROM users WHERE email = ?";
db.query(query, [userEmail], (err, results) => {
    if (err) throw err;
    console.log(results);
});

Always use parameterized queries to prevent SQL Injection attacks effectively and safely.

2. How to Secure APIs with JWT

JSON Web Tokens (JWTs) are a popular way to secure REST APIs by authenticating users and verifying their sessions securely.

Node.js 

const token = jwt.sign({ id: user._id }, process.env.JWT_SECRET, { expiresIn: "1h" });

Always store your secret keys safely and rotate JWTs periodically to enhance API security.

4. Basics of Hashing Passwords

Hashing passwords before storing them in the database adds a security layer against data breaches.

Python 

import bcrypt

password = "mypassword"
hashed = bcrypt.hashpw(password.encode('utf-8'), bcrypt.gensalt())

Always use strong hashing algorithms like bcrypt, scrypt, or Argon2 for password protection.

5. Difference Between var, let and const

JavaScript offers multiple ways to declare variables: var, let, and const. Knowing their differences helps in writing better code.

JavaScript 

var a = 10;    // Function scoped
let b = 20;    // Block scoped
const c = 30;  // Block scoped (constant)

Prefer using let and const to avoid hoisting issues and accidental reassignments.
